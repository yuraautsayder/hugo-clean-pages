Контент: статьи и «О компании»

В этом разделе описаны все аспекты работы с контентом сайта, включая создание статей, управление отзывами и редактирование страницы “О компании”.

Система статей

Структура статей

Все статьи размещаются в директории content/posts/ и автоматически отображаются на главной странице и в соответствующих списках.

Создание новой статьи

1. Создание файла

Создайте новый Markdown файл в content/posts/ с расширением .md, например my-article.md.

2. Фронтматтер (метаданные)

Каждая статья должна содержать фронтматтер в формате TOML:

+++
title = "Заголовок статьи"
date = "2025-02-01"
draft = false
image = "/img/cover.png"     # опционально - путь к изображению
author = "Имя автора"        # опционально - автор статьи
description = "Краткое описание для SEO"  # опционально
tags = ["тег1", "тег2"]      # опционально - теги для категоризации
categories = ["категория"]    # опционально - категории
+++

3. Содержание статьи

После фронтматтера добавьте основной текст статьи в формате Markdown:

# Заголовок статьи

Введение в тему статьи...

## Основная часть

Основной контент статьи с поддержкой всех возможностей Markdown:

- **Жирный текст**
- *Курсив*
- [Ссылки](https://example.com)
- ![Изображения](/img/example.jpg)

### Подзаголовки

Дополнительные разделы статьи...

## Заключение

Итоги и выводы по статье.

4. Изображения для статей

  • Размещайте изображения в static/img/
  • Используйте относительные пути: /img/filename.jpg
  • Рекомендуемые форматы: JPG, PNG, WebP
  • Оптимизируйте размеры для веба

Управление существующими статьями

Редактирование

  1. Откройте файл статьи в content/posts/
  2. Внесите изменения в фронтматтер или содержимое
  3. Сохраните файл
  4. Hugo автоматически пересоберет страницу

Удаление

  1. Удалите файл статьи из content/posts/
  2. Hugo автоматически уберет статью из всех списков

Черновики

Установите draft = true для временного скрытия статьи:

+++
title = "Черновик статьи"
draft = true
+++

Система отзывов

Структура отзывов

Отзывы размещаются в директории content/reviews/ и автоматически отображаются на всех страницах сайта.

Создание нового отзыва

1. Создание файла

Создайте новый Markdown файл в content/reviews/, например client-name.md.

2. Фронтматтер отзыва

+++
title = "Имя клиента"
date = "2025-02-01"
role = "Должность"           # опционально - должность/роль
rating = 5                   # опционально - рейтинг (1-5)
weight = 1                   # опционально - порядок отображения
image = "/img/client.jpg"    # опционально - фото клиента
+++

3. Содержание отзыва

Текст отзыва клиента. Может содержать несколько абзацев.

Дополнительные детали и впечатления от работы с компанией.

4. Управление порядком отзывов

Используйте поле weight для контроля порядка отображения:

  • Меньшие значения отображаются первыми
  • По умолчанию отзывы сортируются по дате

Редактирование отзывов

  1. Откройте файл отзыва в content/reviews/
  2. Внесите изменения в метаданные или текст
  3. Сохраните файл
  4. Изменения автоматически применятся на всех страницах

Страница “О компании”

Редактирование информации

1. Основной файл

Страница “О компании” редактируется через файл content/about.md.

2. Фронтматтер

+++
title = "О компании"
description = "Информация о нашей компании"
layout = "about"
image = "/img/company.jpg"   # опционально - обложка страницы
+++

3. Содержание страницы

# О нашей компании

## Миссия
Описание миссии компании...

## Ценности
- **Качество** - мы стремимся к совершенству
- **Инновации** - постоянно развиваемся
- **Клиентоориентированность** - ставим клиентов на первое место

## История
Рассказ о становлении и развитии компании...

## Команда
Описание ключевых сотрудников и их ролей...

## Контакты
- **Адрес**: ул. Примерная, 123
- **Телефон**: +7 (999) 123-45-67
- **Email**: info@company.com
- **Время работы**: Пн-Пт 9:00-18:00

Кастомизация страницы “О компании”

Изменение дизайна

  1. Отредактируйте шаблон layouts/_default/about.html
  2. Измените стили в assets/css/custom.css
  3. Перезапустите Hugo сервер

Добавление новых секций

  1. Добавьте новые разделы в Markdown
  2. При необходимости создайте дополнительные partials
  3. Обновите CSS для новых элементов

Управление контентом

Организация файлов

content/
├── posts/           # Статьи и новости
│   ├── _index.md    # Описание секции статей
│   ├── article1.md  # Первая статья
│   └── article2.md  # Вторая статья
├── reviews/         # Отзывы клиентов
│   ├── _index.md    # Описание секции отзывов
│   ├── client1.md   # Отзыв первого клиента
│   └── client2.md   # Отзыв второго клиента
├── about.md         # Страница "О компании"
└── docs/            # Документация
    ├── _index.md    # Главная страница документации
    ├── components.md # Компоненты и шаблоны
    ├── content.md   # Этот файл
    ├── reviews.md   # Документация по отзывам
    └── sources.md   # Источники

Лучшие практики

Для статей

  • Используйте описательные заголовки
  • Добавляйте краткие описания для SEO
  • Оптимизируйте изображения
  • Используйте теги и категории для организации

Для отзывов

  • Получайте разрешение на публикацию
  • Сохраняйте аутентичность текста
  • Добавляйте рейтинги для объективности
  • Используйте веса для приоритизации

Для страницы “О компании”

  • Регулярно обновляйте информацию
  • Добавляйте актуальные фотографии
  • Включайте контактную информацию
  • Рассказывайте о достижениях и планах

Автоматизация

  • Hugo автоматически пересобирает страницы при изменении контента
  • Используйте hugo server --watch для автоматического обновления
  • Настройте CI/CD для автоматической публикации изменений
